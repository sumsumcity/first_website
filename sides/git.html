Git Tutorial


Grundlagen

Code Erklärung
git init Macht ein Repository
git clone url ordner Fügt bestehendes Repository hinzu. (url == Link, ornder == Ordnername)
git add datei.txt Macht datei.txt für den commit bereit. (wenn alles -> --all)
git commit -m „Text“ Stellt die geaddeten Dateien online
git log Zeigt die hashes von dem Repository an
git reset --hard HEAD Head wird zum letzten Commit gemacht
git checkout hash Macht alten Commit zum Head (hash == Nummer bei log)
git status Zeigt alle Änderungen seit dem letzten Commit
git rm datei Datei wird gelöst und für den Commit vorbereitet (z.B. datei == hal.txt)
git push Alles online stellen
git pull Sich updaten. Erstellte Dateien von den anderen herunterladen
git fetch ; git status Diese zwei Befehle nacheinander zeigt an ob etwas gepusht wurde

Branch

Code Erklärung
git branch testing Erstellt einen Branch (testing == Name des Branches)
git branch Zeigt alles Branches
git merge testing Fügt den Master und den testing Branche zusammen (testing == Name des Branches)
git branch -d testing Löscht den testing Branch (testing == Name des Branches)
git checkout branchname Wechselt in den anderen Branch (z.B. branchname == testing
